# Quick Deployment Guide

## Deploy to Render (Recommended)

### Option 1: One-Click Blueprint Deployment

1. Push this code to GitHub
2. Go to [Render Dashboard](https://dashboard.render.com)
3. Click "New +" → "Blueprint"
4. Connect your GitHub repository
5. Select `Coatvision/CoatVision_MVP`
6. Click "Apply" - Render will deploy both backend and frontend automatically

### Option 2: Use the CLI

```bash
# Install Render CLI
npm install -g render-cli

# Deploy
render deploy
```

## What Gets Deployed?

- ✅ **Backend API** (FastAPI) - Web Service at `/backend`
- ✅ **Frontend Dashboard** (React/Vite) - Static Site at `/frontend`
- ❌ **Mobile App** - Not deployed to Render (use Expo/App Stores)

## Environment Variables to Set

After deployment, configure these in Render Dashboard:

### Backend Service
- `SECRET_KEY` - Auto-generated by Render
- `OPENAI_API_KEY` - Your OpenAI API key (optional)
- `DATABASE_URL` - Database URL (optional, defaults to SQLite)

### Frontend Site
- `VITE_API_URL` - Auto-set from backend service URL

## Verify Deployment

- Backend health: `https://your-backend.onrender.com/health`
- Backend API: `https://your-backend.onrender.com/`
- Frontend: `https://your-frontend.onrender.com/`

## Troubleshooting

**Build fails?**
- Check logs in Render Dashboard
- Verify all files are committed to Git
- Ensure Python 3.11+ and Node.js 18+ are specified

**VSCode hanging?**
- Deployment is done via Render's platform, not VSCode
- Push code to GitHub, then deploy via Render Dashboard
- No need to deploy from VSCode
